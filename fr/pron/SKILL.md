# Adding a New Mélissa Document

This document describes the full procedure for adding a new Mélissa pronunciation lesson (a French interview/documentary excerpt) to the Pronunciation Practice section.

---

## File locations

Each Mélissa lesson lives in its own numbered subfolder under `fr/pron/Melissa/`.

| File | Path |
|---|---|
| Lesson HTML | `fr/pron/Melissa/NN/index.html` (`NN` = 2-digit lesson number, e.g. `01`, `02`) |
| Audio files (CDN) | `https://languages.rmlives.com/rfi/Melissa/NN/` |
| Navigation script | `assets/js/nav-pron.js` |
| Source transcript | Provided as French text, split into short chunks, plus an English translation |

---

## Step 1 — Prepare the audio

Audio for each lesson lives on Cloudflare/rmlives at `https://languages.rmlives.com/rfi/Melissa/NN/`:

- `audio.mp3` — the full, uncut interview clip (used by the draggable audio player)
- `01.mp3`, `02.mp3`, `03.mp3`, … — one file per chunk/glossary word, named with **just the zero-padded chunk number**, no descriptive text in the filename

Do not hand-write a `<audio>` tag per chunk with the French text baked into the filename — the real files on the server are plain numbers. Audio elements are generated by a small script instead (see Step 3).

---

## Step 2 — Create `Melissa/NN/index.html`

Copy `Melissa/01/index.html` as a template. Update:

- `<h1>` → the interview's title/topic
- `nav-pron.js?v=` → leave as-is for now; the final value is set in Step 5
- `#audioElement` `src` → `https://languages.rmlives.com/rfi/Melissa/NN/audio.mp3`

### Main text (`#main-text`)

Split the French transcript into short clickable chunks:

```html
<span class="chunk" onclick="playAudio('audio1')"> [French phrase]</span>
<span class="chunk" onclick="playAudio('audio2')"> [French phrase]</span>
...
```

Number chunks sequentially starting at 1. After the last chunk, add an `<hr />` and a single `<p class="text-content"><span>[full English translation]</span></p>` block.

### Glossary (`#glossary`)

Continue the audio numbering **from where the main-text chunks left off** (e.g. if the transcript used `audio1`–`audio49`, the glossary starts at `audio50`). Each row:

```html
<tr>
  <td class="words" onclick="playAudio('audioNN')">[French word] /[IPA]/</td>
  <td>[English meaning]</td>
</tr>
```

The final audio number used in the glossary is the total chunk count for Step 3.

---

## Step 3 — Generate the hidden audio elements

Instead of hand-writing one `<audio>` tag per chunk/glossary word, generate them with a script. Replace the "Hidden Audio Elements" block with:

```html
<!-- Hidden Audio Elements (generated by script below) -->
<div id="hiddenAudioContainer"></div>
<script>
  (function () {
    const base = "https://languages.rmlives.com/rfi/Melissa/NN/";
    const container = document.getElementById("hiddenAudioContainer");
    for (let i = 1; i <= TOTAL; i++) {
      const num = String(i).padStart(2, "0");
      const audio = document.createElement("audio");
      audio.id = "audio" + i;
      audio.src = base + num + ".mp3";
      container.appendChild(audio);
    }
  })();
</script>
```

Set `base` to the lesson's own folder and `TOTAL` to the highest `audioN` id used across the main text and glossary (i.e. the total number of `.mp3` files for that lesson).

---

## Step 4 — Update `nav-pron.js`

Mélissa entries are **not** a separate sidebar section — they live inside the existing `RFI News` accordion, grouped by year alongside the RFI entries, in chronological order. Add a new link to the matching year's `sidebar-submenu` (create a new year dropdown, copying the pattern of the `2024`/`2025`/`2026` blocks, if none exists yet):

```html
<a href="/fr/pron/Melissa/NN/index.html" class="sidebar-link">
  <i class="bi bi-calendar"></i> Mélissa MM/DD/YY
</a>
```

Label format: `Mélissa MM/DD/YY` (the date the lesson was added, e.g. `Mélissa 07/13/26`) — matches the `RFI MM/DD/YY` label style used for RFI entries. All hrefs in `nav-pron.js` are **root-relative** (`/fr/pron/...`), not page-relative — this matters because Mélissa pages live one folder deeper (`fr/pron/Melissa/NN/`) than RFI pages (`fr/pron/RFIyymmdd/`), so page-relative links (`../...`) would resolve to the wrong directory depth on Mélissa pages.

---

## Step 5 — Bump the `nav-pron.js` version in all pronunciation HTMLs

Every time `nav-pron.js` changes, every HTML file that loads it must get a new `?v=` query string so browsers don't serve a cached copy. The version format is `yyMMddHHmm`.

```powershell
$v = Get-Date -Format "yyMMddHHmm"
$folder = "c:\Users\Owner\Desktop\MAIN\WEBSITE\My Website\languages\fr\pron"
Get-ChildItem "$folder\*.html" -Recurse | ForEach-Object {
    $c = Get-Content $_.FullName -Raw -Encoding UTF8
    $u = $c -replace 'nav-pron\.js\?v=[^\s"]+', "nav-pron.js?v=$v"
    if ($u -ne $c) { [System.IO.File]::WriteAllText($_.FullName, $u, [System.Text.Encoding]::UTF8) }
}
```

This must be run **after every** change to `nav-pron.js`, not just when adding a new lesson.

---

## Step 6 — Update `sitemap.html`

In `sitemap.html`, find the `"French"` node (under Pronunciation) in `graphData.nodes` and point its `url` at the newest pronunciation lesson added — this may be the latest Mélissa lesson or the latest RFI entry, whichever was added most recently:

```js
{
  id: "French",
  group: 2,
  level: 1,
  url: "fr/pron/Melissa/NN/index.html",
},
```

There is only one entry to change — it always links to the single most recently added lesson across both Mélissa and RFI.

## Checklist

- [ ] `Melissa/NN/index.html` created with numbered clickable chunks, English translation, and glossary
- [ ] All French characters are correct UTF-8
- [ ] Hidden audio elements generated via the loop script (Step 3), not hand-written per file, pointing at `https://languages.rmlives.com/rfi/Melissa/NN/`
- [ ] `nav-pron.js` updated: new link added inside the correct year's RFI News dropdown, labeled `Mélissa MM/DD/YY`, using a root-relative href
- [ ] `nav-pron.js?v=` bumped in **all** pronunciation HTML files (Step 5 PowerShell)
- [ ] `sitemap.html`'s `"French"` node updated to the newest lesson
